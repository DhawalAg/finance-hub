# REPL Flow — Design Notes

> Working notes on the interactive research session. See [[50-projects/second-brain/main/06-user-flows#5b Flow 3b Interactive Research Session REPL]] for the spec.

## Core Commands

- `compare 1 3 5` — side-by-side comparison with LLM reasoning
- `skim 3` — fetch full content, LLM-generated summary
- `queue 1 5` — save for later processing
- `refine "graph memory vs summary memory"` — narrow search with context
- `add 1 2 3` or `add github` — select results for extraction/ingestion
- Free-form text — follow-up questions with session context

## Assisted Search Flow

`brain search --assist "vague topic"` — planner agent asks 2-3 clarifying questions before executing. Opt-in, not default.

## Session State Model

The session needs to hold:
- Original query + sub-queries generated
- All search results with scores
- User selections (compared, skimmed, queued, added)
- Conversation context for free-form follow-ups
- Source configuration for the session

Key principle: session state is ephemeral (in-memory). Persistent artifacts (queued items, ingested claims) write to vault/SQLite on user action.

## Quick Interaction Patterns

Users who know what they want should be fast:
- Search → see results → `add 1 2 3` → done (5 seconds)
- Search → `add github` (all GitHub results) → done
- Search → `compare 1 3` → `add 1` → done

Don't force deep interaction. The REPL is a convenience layer, not a gate.

## Open Questions

1. How does readline/prompt library handle multi-line output + input? (inquirer vs raw readline vs ink)
2. Should `refine` re-score all results or just fetch new ones?
3. How to handle session timeout / accidental exit? (Save session state to tmp file?)
4. Should free-form text trigger a new search or use existing results as context?

## From Steelman

- fabric's composability: make each command a composable unit
- REPL should be a thin wrapper over one-shot commands, not a separate system
- One-shot + piping for automation, REPL for exploration
- Build one-shot first (A1-A3), REPL as session wrapper (A5)
